# Менториум — Telegram Bot

Telegram-бот воронка для сервиса репетиторов по математике «Менториум».

## Возможности

- 3 общих экрана (приветствие, выбор цели, выбор роли)
- 8 контентных веток (4 для родителей + 4 для школьников)
- Nurture-цепочки для каждой ветки (4 шага с задержками)
- Постоянное меню: «Записаться на пробное» / «Написать менеджеру»
- CTA-экраны с перенаправлением к менеджеру
- Аналитика просмотров экранов и кликов
- Админ-панель для рассылок (в следующем блоке)

## Требования

- Python 3.10+
- pip

## Установка и запуск

```bash
# 1. Клонировать репозиторий
git clone <repo_url>
cd mentorium_bot

# 2. Создать виртуальное окружение
python -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate   # Windows

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Настроить переменные окружения
cp .env.example .env
# Отредактировать .env — вставить BOT_TOKEN, ADMIN_IDS, MANAGER_CONTACT_LINK

# 5. Запустить бота
python run.py
```

## Структура проекта

```
mentorium_bot/
├── run.py                     # Точка входа
├── bot/
│   ├── config.py              # Конфигурация из .env
│   ├── loader.py              # Инициализация Bot + Dispatcher
│   ├── db/                    # База данных (SQLite)
│   ├── content/               # YAML-файлы с контентом экранов
│   ├── services/              # Бизнес-логика
│   ├── handlers/              # Обработчики сообщений и callback
│   ├── keyboards/             # Генерация клавиатур
│   ├── middlewares/           # Мидлвари aiogram
│   ├── scheduler/             # APScheduler задачи
│   └── utils/                 # Утилиты
└── tests/
```

## Контент

Все тексты экранов хранятся в `bot/content/*.yaml`.
Изменение текстов **не требует** правки Python-кода — достаточно отредактировать YAML и перезапустить бота.

## Переменные окружения

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен Telegram-бота |
| `ADMIN_IDS` | ID админов через запятую |
| `MANAGER_CONTACT_LINK` | Ссылка на менеджера |
| `DATABASE_PATH` | Путь к файлу SQLite |
| `LOG_LEVEL` | Уровень логирования |
